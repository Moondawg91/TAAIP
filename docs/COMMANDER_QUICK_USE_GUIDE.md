# Commander Quick-Use Guide

Purpose:
- Run a command decision walkthrough using the live, connected workflow.
- Use authoritative outputs only.

## Access Perspective

Open the commander shell as commander perspective:

- default commander perspective, or
- add `?role=commander` to the shell URL

Commander path:
- Workflow Home
- Command Center
- Mission Adjustment
- Market to ROI Diagnostics
- TWG and Targeting Board
- Asset, Execution, Processing
- Power BI

## Commander Workflow Steps

## 1) Command Center

Focus on:
- command summary and connected phase2 signals
- LOE posture
- accountability and execution posture

Command question to answer:
- What is the current command picture and where is risk concentrated?

## 2) Mission Adjustment

Focus on:
- mission adjustment recommendation and confidence
- rationale and evidence references

Command question to answer:
- What mission movement is supported by current evidence?

## 3) Diagnostics

Focus on:
- market, funnel, school access, school plan, ROI linkage

Command question to answer:
- Which drivers explain the recommendation and where can we intervene?

## 4) TWG and Targeting Board

Focus on:
- prioritized actions
- board posture and decision linkage

Command question to answer:
- Which actions are approved and what downstream effect is expected?

## 5) Asset / Execution / Processing

Focus on:
- execution tracker status
- blocked/off-track items and escalations
- flash-to-bang health

Command question to answer:
- What is blocked now and what was escalated?

## 6) Power BI Surface

Focus on:
- export-facing operational view for command reporting

Command question to answer:
- Does exported reporting match the same pipeline shown in command workflow?

## No-Data Handling (Commander Talking Points)

- No-data scopes are surfaced honestly; no fabricated defaults are shown.
- Partial-signal states are warnings to narrow scope or refresh authoritative sources.
- Admin refresh is separate; commander flow does not expose refresh controls.
